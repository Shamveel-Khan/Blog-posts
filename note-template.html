Cross-Site Scripting (XSS) Attacks

Last updated: July 3, 2025 ‚Ä¢ Reading time: 0 min

Introduction to XSS

Cross-Site Scripting (XSS) attacks are a type of injection where malicious scripts are injected into otherwise benign and trusted websites. XSS attacks occur when an attacker uses a web application to send malicious code, generally in the form of a browser side script, to a different end user.

Types of XSS Attacks

1. Stored XSS

The malicious script is permanently stored on the target servers (in a database, message forum, visitor log, etc.). The victim then retrieves the malicious script from the server when it requests the stored information.

2. Reflected XSS

The malicious script comes from the current HTTP request. The injected script is reflected off the web server, such as in an error message, search result, or any response that includes some or all of the input sent to the server as part of the request.

3. DOM-based XSS

The vulnerability exists in client-side code rather than server-side code. The attacker's payload is executed as a result of modifying the DOM environment in the victim's browser.

Common XSS Payloads

Here are some common XSS payloads that attackers use:

<script>alert('XSS')</script>
<img src=x onerror=alert('XSS')>
<svg onload=alert('XSS')></svg>
<body onload=alert('XSS')>

Prevention Techniques

Input Sanitization

Always sanitize user input. Remove or encode any potentially dangerous characters. Use libraries like DOMPurify to sanitize HTML.

Content Security Policy (CSP)

Implement a strong Content Security Policy to prevent the execution of unauthorized scripts. Example CSP header: 

Content-Security-Policy: default-src 'self'; script-src 'self' https://trusted.cdn.com; object-src 'none';

HTTPOnly Cookies

Mark cookies as HTTPOnly to prevent them from being accessed by client-side scripts. This helps mitigate the impact of XSS attacks. 

// Setting HTTPOnly cookie in Express.js
res.cookie('session', 'value', { 
    httpOnly: true,
    secure: true,
    sameSite: 'strict'
});

X-XSS-Protection Header

Although deprecated in modern browsers, you can still set this header for older browsers: 

X-XSS-Protection: 1; mode=block

Testing for XSS Vulnerabilities

Use these tools and techniques to test for XSS vulnerabilities:

  • Manual testing with various payloads
  • Burp Suite's Scanner
  • OWASP ZAP
  • XSSer automation tool
  • Browser extensions like XSS Hunter

Best Practice:

Always treat user input as untrusted. Implement multiple layers of defense including input validation, output encoding, and proper security headers.
