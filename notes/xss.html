üß† INTRO

XSS happens when user input is treated as JavaScript or HTML.
As a result, through the user's input, JavaScript can be executed ‚Äî which can expose cookies or session IDs.



üí£ TYPES OF XSS

üîÅ Reflected XSS

This happens when you put in an input, and then the front end sends that to the back end. The response from the back end contains the JavaScript, which is executed as soon as the browser parses it.



üíæ Stored XSS

This happens when you insert a malicious payload containing JavaScript, and it's stored on the server (like in a comment or post).
Whenever other users load that page, the JS is executed, exposing their cookies or session data.



üß© DOM XSS

The payload is executed directly by the front end, without any response from the server.
This happens because of unsanitized client-side inputs.
